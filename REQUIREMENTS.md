# sush.sh (sushi) v0.1 需求文档

## 项目概述

sush.sh 是一款轻量级终端 SSH + SFTP 统一管理工具，基于 TUI 界面，单文件分发，跨平台支持（macOS / Linux / Windows）。

**核心价值**：统一的 SSH 凭证管理 + SSH/SFTP 无缝切换体验，解决现有工具 SSH 与 SFTP 割裂的痛点。

**名称**：sush.sh，简称 sushi

## v0.1 功能范围

### F1: 自动导入 SSH Config

**描述**：首次启动时自动读取 `~/.ssh/config`，解析为主机列表。

**规则**：
- 解析字段：`Host`、`HostName`、`User`、`Port`、`IdentityFile`、`ProxyJump`
- 忽略通配符 Host（如 `Host *`）
- 支持多个 `IdentityFile` 声明
- 启动时自动检测 `~/.ssh/config` 变更，增量同步（新增/更新，不删除用户手动添加的主机）
- 导入的主机标记来源为 `ssh_config`，与用户手动添加的主机区分

**验证标准**：
- 启动后能看到 `~/.ssh/config` 中定义的所有非通配符主机
- 修改 ssh config 后重启能同步变更

### F2: 主界面 — 搜索 + 主机列表

**描述**：极简主界面，顶部搜索框 + 主机列表 + 底部快捷键提示。

**布局**：
```
┌─ sushi ────────────────────────────────────────┐
│                                                │
│  > 搜索...  █                                   │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │ ● prod-web-01   192.168.1.10      web   │  │
│  │   prod-db-01    192.168.1.20       db   │  │
│  │   dev-server    10.0.0.5          dev   │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  Enter:SSH  F2:SFTP  F5:新建  ?:帮助  Q:退出    │
└────────────────────────────────────────────────┘
```

**搜索规则**：
- 多维模糊搜索：匹配 Host 别名、HostName/IP、User、标签、描述
- 输入即搜，实时过滤
- 空搜索显示全部主机
- 搜索结果高亮匹配字段

**主机列表**：
- 显示：别名、地址、标签
- 上下键导航，选中行高亮
- 支持标签（扁平标签，非分组，一个主机可有多个标签）

**快捷键提示栏**：
- 固定在底部一行
- 根据当前上下文动态显示可用操作
- 格式：`键:动作` 空格分隔

### F3: SSH 连接 — 接管模式

**描述**：选中主机后按 Enter 进入 SSH 连接，sushi 进入 I/O 转发模式（前缀键多路复用）。

**连接流程**：
1. 从主机配置读取连接参数
2. 认证顺序：ssh-agent → IdentityFile（自动解密提示密码）→ 密码认证（TUI 弹出密码输入框）
3. 连接成功后进入接管模式
4. 连接失败显示错误信息，返回主界面

**接管模式行为**：
- sushi 持有 I/O 通道，将 stdin 字节流转发给远程 PTY，远程输出转发到 stdout
- 远程程序（vim、tmux、htop 等）完全兼容
- 终端 resize 事件转发给远程
- `exit` / `Ctrl-D` 退出远程 shell → 自动返回 sushi 主界面

**前缀键 `Ctrl-Space`**：
- SSH 模式下按 `Ctrl-Space` → 切换到 SFTP 面板（SSH 会话保持）
- SFTP 面板下按 `Ctrl-Space` → 切回 SSH 会话
- 按下 `Ctrl-Space` 后有短暂提示，避免用户不知道发生了什么

### F4: SFTP 文件浏览器 — 单面板

**描述**：单面板文件浏览器，可在本地/远程目录间切换。

**布局**：
```
┌─ sushi ─ SFTP: prod-web-01 ────────────────────┐
│ 远程: /var/www/html/                            │
│ ┌──────────────────────────────────────────────┐│
│ │ ../                                          ││
│ │ ▸ assets/                          <DIR>     ││
│ │ ▸ css/                             <DIR>     ││
│ │   index.html                       2.4 KB    ││
│ │   style.css                        1.1 KB    ││
│ │   app.js                          12.8 KB    ││
│ └──────────────────────────────────────────────┘│
│                                                 │
│ Tab:本地/远程 F5:下载 F6:上传 Ctrl-Space:SSH Q:返回│
└─────────────────────────────────────────────────┘
```

**浏览行为**：
- 默认打开远程用户 `$HOME`，失败回落到 `/`
- `Tab` 键切换本地/远程视图
- 本地视图默认打开 sushi 运行时的工作目录（`$PWD`）
- 目录排前，文件排后，各自按名称排序
- 显示文件大小（人类可读格式）、目录标记
- Enter 进入目录 / `..` 返回上级
- 支持多选（Space 标记）

### F5: 文件上传/下载 + 进度显示

**描述**：在 SFTP 面板中上传/下载文件，显示实时进度。

**操作方式**：
- 远程视图中选中文件 → `F5` 下载到本地当前目录
- 本地视图中选中文件 → `F6` 上传到远程当前目录
- 多选后批量传输

**进度显示**：
- 底部显示传输进度条
- 格式：`[████████░░ 78%] uploading style.css  1.8MB/2.4MB`
- 传输完成后短暂提示成功，然后恢复快捷键栏
- 传输失败显示错误信息

**v0.1 限制**：
- 仅支持单文件传输（不支持文件夹递归传输）
- 不支持断点续传
- 不支持后台传输（传输时阻塞操作）

### F6: 双 Ctrl-C 返回主界面

**描述**：在 SFTP 面板中，快速双击 `Ctrl-C`（500ms 内）返回主界面。

**作用域**：
- **SFTP 面板**：双 `Ctrl-C` → 断开连接，返回主界面
- **SSH 接管模式**：`Ctrl-C` 始终透传给远程进程，不拦截。退出方式为 `exit` / `Ctrl-D`
- **主界面**：单次 `Ctrl-C` 清空搜索框；搜索框为空时 `Ctrl-C` 退出程序
- 正在传输时第一次 `Ctrl-C` 取消传输，第二次返回主界面

## 非功能需求

### 性能
- 启动时间 < 200ms（含 ssh config 解析）
- 搜索响应 < 50ms
- 内存占用 < 30MB（空闲状态）

### 分发
- 单文件二进制，无运行时依赖
- GitHub Release 提供 macOS (arm64/x86_64)、Linux (x86_64)、Windows (x86_64) 预编译包

### 兼容性
- 最低终端要求：支持 256 色
- 支持 UTF-8 终端

## 术语表

| 术语 | 含义 |
|------|------|
| 接管模式 | sushi 持有 I/O 通道，转发 stdin/stdout 到远程 PTY，同时拦截前缀键 |
| 前缀键 | `Ctrl-Space`，用于在 SSH/SFTP 模式间切换 |
| 单面板 | 同一时间只显示一个目录（本地或远程），通过 Tab 切换 |
