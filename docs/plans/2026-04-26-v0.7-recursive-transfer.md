# v0.7 Recursive Transfer Implementation Plan

> **For Claude:** Use `${SUPERPOWERS_SKILLS_ROOT}/skills/collaboration/executing-plans/SKILL.md` to implement this plan task-by-task.

**Goal:** Add recursive directory upload and download in the SFTP browser with conflict prompts, resumable per-file decisions, and aggregate progress.

**Architecture:** Keep recursive transfer inside the existing SFTP workflow instead of introducing a new transfer subsystem. `sftp/transfer.rs` becomes the execution engine for directory scan + per-file transfer, while `App` owns the interactive decisions for file conflicts and per-file failures through modal prompts and a single foreground task state.

**Tech Stack:** Rust 1.95, tokio, existing `russh-sftp` client/session, existing `App` state machine, existing progress bar, TUI confirm dialog

---

## Scope

- Upload/download directories recursively from the existing SFTP browser
- Preserve the selected directory itself at the destination
- Prompt on file conflicts with `Skip` or `Overwrite`
- Support "apply to remaining conflicts" for file conflicts
- Show aggregate progress: `current file index / total files` plus current file byte progress
- Pause on per-file transfer failures and let user choose `Retry`, `Skip`, or `Cancel task`

## Non-Goals

- No background transfer queue
- No multi-task parallel transfer
- No resumable byte-range transfer
- No conflict prompt for directories
- No dual-pane redesign
