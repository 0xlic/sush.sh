# v0.7 Transfer Queue and Background Transfer Implementation Plan

> **For Claude:** Use `${SUPERPOWERS_SKILLS_ROOT}/skills/collaboration/executing-plans/SKILL.md` to implement this plan task-by-task.

**Goal:** Add a current-connection-scoped global transfer queue so upload and download tasks continue sequentially in the background, while a compact transfer indicator remains visible in Main, SSH, and SFTP views.

**Architecture:** Keep one active transfer plus a FIFO queue in `App`, and enqueue already-expanded transfer jobs instead of re-reading UI state later. Replace the full-width bottom progress bar with a shared compact transfer badge rendered alongside each screen's existing hints, so background transfer state stays visible without taking over the entire status line.

**Tech Stack:** Rust 1.95, tokio, ratatui, crossterm, existing `App` state machine, existing SFTP transfer helpers, existing main/ssh/sftp views, existing `StatusBar` widget

---

## Scope

- Add one FIFO transfer queue scoped to the current active connection
- Support background sequential execution while switching between Main, SSH, and SFTP views
- Show a compact right-aligned transfer badge in Main, SSH, and SFTP status bars
- Badge format is `↑ 2/10 37%` or `↓ 2/10 37%`
- Keep existing left-side action hints visible while transfers run
- Clear queue state when the current connection is closed

## Non-Goals

- No separate queue panel
- No parallel transfer execution
- No cross-connection persistence
- No filename, speed, ETA, or expandable queue details in the badge
- No resumable transfers
