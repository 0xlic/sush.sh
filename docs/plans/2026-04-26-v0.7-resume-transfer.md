# v0.7 Resume Transfer Implementation Plan

> **For Claude:** Use `${SUPERPOWERS_SKILLS_ROOT}/skills/collaboration/executing-plans/SKILL.md` to implement this plan task-by-task.

**Goal:** Add minimal resume support for normal file upload/download so repeated transfers continue from the already existing byte offset instead of restarting from zero.

**Architecture:** Keep the existing queue and recursive runtime model unchanged. Extend the low-level single-file transfer helpers in `src/sftp/transfer.rs` to detect an existing partial target, seek/skip to the matching byte offset, and continue transferring only the remaining bytes. Recursive and queued transfers inherit resume automatically because they already reuse the same single-file helpers.

**Tech Stack:** Rust 1.95, tokio, russh-sftp, existing `App` queue runtime, existing transfer progress channel
