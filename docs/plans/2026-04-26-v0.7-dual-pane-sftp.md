# v0.7 Dual-Pane SFTP Implementation Plan

> **For Claude:** Use `${SUPERPOWERS_SKILLS_ROOT}/skills/collaboration/executing-plans/SKILL.md` to implement this plan task-by-task.

**Goal:** Add an adaptive dual-pane SFTP layout that shows local and remote panes side by side on wide terminals and falls back to the active pane on narrow terminals.

**Architecture:** Keep the current `SftpPaneState` as the single source of truth for both local and remote directory data, but split "which side is active" from "how the UI is rendered". The TUI view decides whether to render one pane or two based on terminal width, while `App` keeps key handling focused on the active pane only.

**Tech Stack:** Rust 1.95, ratatui, crossterm, existing `App` state machine, existing `SftpPaneState`, existing transfer/progress widgets

---

## Scope

- Wide terminal: render local pane on the left and remote pane on the right
- Narrow terminal: render only the active pane
- `Tab`: switch the active pane only
- Each pane keeps its own selected row while focus moves between panes
- Existing actions (`Enter`, `d`, `u`, `e`, `Ctrl-\`, transfer progress, conflict/failure dialogs) continue to work

## Non-Goals

- No manual toggle between single-pane and dual-pane modes
- No resizable split ratio
- No persisted layout preference
- No new transfer queue or drag-and-drop interactions
- No multi-select in this task

fn tab_switches_active_pane_without_resetting_local_selection() {
    let mut app = app_with(vec![]);
    let mut pane = SftpPaneState::new("/remote".into());
    pane.side = PaneSide::Local;
    pane.local_entries = vec![
        file_entry("a.txt", false),
        file_entry("b.txt", false),

    app.mode = AppMode::Sftp;

    app.handle_key(KeyEvent::from(KeyCode::Tab));

    assert_eq!(pane.side, PaneSide::Local);
    assert_eq!(pane.local_list_state.selected(), Some(1));
