# v0.7 SFTP Multi-Select Implementation Plan

> **For Claude:** Use `${SUPERPOWERS_SKILLS_ROOT}/skills/collaboration/executing-plans/SKILL.md` to implement this plan task-by-task.

**Goal:** Add local and remote SFTP multi-select with anchor/range selection, multi-select mode UI, and batch transfer/delete actions.

**Architecture:** Keep selection state inside `SftpPaneState` so local and remote panes each own their own selected set, anchor, and multi-select mode. `App` keeps key handling and batch action dispatch focused on the active pane, while the SFTP view switches its status/help presentation when the active pane has an active multi-selection.

**Tech Stack:** Rust 1.95, ratatui, crossterm, existing `App` state machine, existing `SftpPaneState`, existing SFTP transfer and confirm-dialog flow

---

## Scope

- Local pane supports multi-select
- Remote pane supports multi-select
- `Space` toggles current item and updates the anchor
- Double-press `Space` on another item with an existing anchor selects the inclusive range between anchor and current row
- Range selection works both upward and downward
- `Esc` clears multi-select state for the active pane
- Active multi-select mode shows only batch actions relevant to the current pane
- After executing a batch action, clear multi-select state for that pane

## Non-Goals

- No cross-pane mixed selection
- No persistent selection across directory changes
- No background queue in this task
- No Shift-based selection shortcuts
- No drag or mouse support
