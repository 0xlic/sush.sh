# v0.7 Remote Edit Implementation Plan

> **For Claude:** Use `${SUPERPOWERS_SKILLS_ROOT}/skills/collaboration/executing-plans/SKILL.md` to implement this plan task-by-task.

**Goal:** Add remote file editing from the SFTP view by opening a downloaded temp file in the operating system's default GUI application and auto-uploading on save.

**Architecture:** Keep the existing `App` mode model and implement remote editing as a lightweight session owned by `App`, triggered from the SFTP key handler. Download and upload stay in the SFTP layer, platform-specific opening is isolated behind a small launcher abstraction, and save detection uses file watching with polling and content fingerprint fallback.

**Tech Stack:** Rust 1.95, tokio, existing `App` state machine, `russh-sftp`, `tempfile`, a small file-watch crate if needed, existing ratatui status bar

---

## Scope

- Single remote file only
- Trigger from SFTP remote pane with `e`
- Open with platform default GUI application
- Auto-upload on save
- Support macOS, Linux, Windows command selection

## Non-Goals

- No built-in editor
- No conflict merge UI
- No multi-file edit sessions
- No directory sync
- No resume after application restart

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct OpenCommand {
    pub program: String,
    pub args: Vec<String>,
}

impl OpenCommand {
    pub fn for_platform(platform: &str, path: &str) -> Self {
        match platform {
            "macos" => Self {
                program: "open".into(),
                args: vec![path.into()],
            },
            "windows" => Self {
                program: "cmd".into(),
                args: vec!["/C".into(), "start".into(), "".into(), path.into()],
            },
            _ => Self {
                program: "xdg-open".into(),
                args: vec![path.into()],
            },
        }
    }
}
```
