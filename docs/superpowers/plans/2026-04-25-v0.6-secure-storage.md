# v0.6 Secure Storage Implementation Plan

> **For Claude:** Use `${SUPERPOWERS_SKILLS_ROOT}/skills/collaboration/executing-plans/SKILL.md` to implement this plan task-by-task.

**Goal:** 为 `sush` 增加基于系统原生安全存储的凭证保存能力，覆盖登录密码与私钥口令，并在 Linux 无可用系统钥匙串时禁止保存、仅允许临时输入并给出明确提示。

**Architecture:** 保持 `hosts.toml` 只存非敏感主机信息，新增独立的 `SecretStore` 抽象来屏蔽各平台系统安全能力。认证流程仍保持 `ssh-agent -> 无口令私钥 -> 需要时再取 secret/手工输入`，并且只在认证成功后静默保存凭证；保存失败不影响当次连接，只把失败原因写入本地 metadata，供下次再次输入时提示。

**Tech Stack:** Rust 1.95, tokio, anyhow, serde/toml, 现有 `App` 状态机, 系统级安全存储封装 crate（待在实现阶段选型并最小接入）

## 约束与决策

- 不引入应用级 `master password`
- 不把明文密码或私钥口令写入 `hosts.toml`
- 只在认证成功后静默保存凭证
- 保存失败不影响本次已成功连接
- Linux 无可用系统钥匙串时：
  - 禁止保存凭证
  - 允许本次临时输入
  - 下次再次需要输入时提示上次未保存的原因，并建议用户安装相应能力
- 优先保持现有认证顺序，不重构无关代码

```rust
mod tests {

    fn entry_key_is_stable() {

        let mut store = SecretStore::new(Box::new(FakeBackend::unavailable()));
        let key = SecretKey::new("host-1", SecretKind::LoginPassword, None);

```

**Step 3: 写最小实现**

在 `src/config/secrets.rs` 中实现：

- `SecretKind`
- `SecretKey`
- `SecretError`
- `SecretBackend` trait
- `SecretStore`
- 仅供测试使用的 `FakeBackend`

推荐的最小接口：

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum SecretKind {
    LoginPassword,
    KeyPassphrase,
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct SecretKey {
    pub service: String,
    pub account: String,
}

pub trait SecretBackend: Send + Sync {
    fn is_available(&self) -> bool;
    fn get(&self, key: &SecretKey) -> Result<Option<String>, SecretError>;
    fn set(&self, key: &SecretKey, value: &str) -> Result<(), SecretError>;
    fn delete(&self, key: &SecretKey) -> Result<(), SecretError>;
}
```

`SecretKey::new()` 规则：

- `service` 固定为 `sush`
- 登录密码：`{host_id}:login_password`
- 私钥口令：`{host_id}:key_passphrase:{identity_hint}`
- `identity_hint` 先用原始路径字符串；后续如需改哈希再单独演进，不在 v0.6 提前抽象

```rust
#[test]
fn secret_save_failures_roundtrip() {
    let dir = TempDir::new().unwrap();
    let path = dir.path().join("hosts.toml");
```

**Step 3: 写最小实现**

在 `src/config/store.rs` 增加：

```rust
#[derive(Debug, Serialize, Deserialize, Default, Clone)]
pub struct SecretSaveFailure {
    pub account: String,
    pub reason: String,
}
```

并在 `Metadata` 中增加：

```rust
#[serde(default)]
pub secret_save_failures: Vec<SecretSaveFailure>,
```

只保存：

- `account`
- 用户可读 `reason`

- 明文 secret
- 原始异常堆栈

**Step 4: 增加最小 helper**

在 `store.rs` 增加便于 `App` 调用的最小方法：

```rust
impl Metadata {
    pub fn upsert_secret_failure(&mut self, account: String, reason: String) { /* ... */ }
    pub fn take_secret_failure(&mut self, account: &str) -> Option<SecretSaveFailure> { /* ... */ }
}
```

语义：

- `upsert` 覆盖同一 account 的旧失败原因
- `take` 在提示后移除，避免每次都重复提示
