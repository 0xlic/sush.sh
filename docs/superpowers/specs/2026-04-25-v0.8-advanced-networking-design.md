# v0.8 高级网络功能设计文档

日期：2026-04-25

## 功能范围

- SSH 端口转发管理（本地 / 远程 / 动态）
- ProxyJump 单跳跳板支持（russh 层真正实现）
- SOCKS5 动态转发（daemon 对外暴露 SOCKS5 代理端口）
- 隧道状态可视化（运行中 / 重连中 / 错误）

## 核心设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 转发进程模型 | 单一 sush daemon 进程 | 自包含，无外部依赖，集中管理所有连接 |
| IPC 机制 | Unix socket（Mac/Linux），Windows 推迟到 v1.0 | 简单可靠，tokio 原生支持 |
| ProxyJump 范围 | 单跳（A → 跳板 → 目标） | 满足主流场景，实现复杂度可控 |
| SOCKS5 方向 | 动态转发（daemon 作为 SOCKS5 服务端） | 用户场景是通过远程主机代理本地流量 |
| 隧道可视化 | 状态列表（TUI 内轮询刷新） | 简单直接，无需独立监控面板 |

## 整体架构

```
用户终端
  │
  ├── sush (TUI 模式)
  │     ├── 主界面  →  p 键  →  ForwardingView（双面板转发管理）
  │     └── 通过 Unix socket 向 daemon 发命令（start/stop/status）
  │
  └── sush --daemon（后台 daemon 模式，同一个二进制）
        ├── 监听 ~/.config/sush/daemon.sock
        ├── 读取 hosts.toml 中各主机的转发规则
        ├── 用 russh 建立 SSH 连接并保持转发通道
        └── 每个活跃规则：russh channel → 本地 TCP 监听端口
```

### Daemon 启动流程

1. 检查 `daemon.sock` 是否已存在且有响应；若已有 daemon 则退出（避免重复启动）
2. 写入 `~/.config/sush/daemon.pid`
3. 监听 `~/.config/sush/daemon.sock`
4. 读取 `hosts.toml`，对所有 `auto_start = true` 的规则建立 SSH 连接并激活转发
5. 进入事件循环：处理 socket 命令、监控 SSH channel 状态、执行断线重连

### TUI 与 Daemon 交互

TUI 通过 `Tick` 每秒轮询一次 `status` 命令（仅在 `ForwardingView` 激活时）。若 TUI 发送命令时 daemon 未运行，自动 spawn `sush --daemon` 后重试。

## IPC 协议

JSON over Unix socket，单次 request/response。

| 命令 | 说明 |
|------|------|
| `{"cmd":"status"}` | 返回所有规则的运行状态列表 |
| `{"cmd":"start","forward_id":"id"}` | 启动指定规则 |
| `{"cmd":"stop","forward_id":"id"}` | 停止指定规则 |
| `{"cmd":"stop_all"}` | 停止所有规则（用户主动关闭 daemon） |

Status 响应示例：
```json
{
  "forwards": [
    {
      "id": "fwd-uuid-1",
      "host_id": "prod-web-01",
      "state": "Running",
      "error": null
    },
    {
      "id": "fwd-uuid-2",
      "host_id": "prod-web-01",
      "state": "Reconnecting",
      "retry_count": 2,
      "max_retries": 5,
      "error": null
    }
  ]
}
```

## 数据模型

`Host` 新增 `forwards` 字段，向后兼容（缺失时默认空数组）：

```toml
[[hosts]]
id = "prod-web-01"
alias = "prod-web-01"
hostname = "192.168.1.10"
port = 22
user = "deploy"
# ... 现有字段不变 ...

[[hosts.forwards]]
id = "fwd-uuid-1"
name = "Web Admin"
kind = "local"           # local | remote | dynamic
local_port = 8080
remote_host = "localhost"  # dynamic 时忽略
remote_port = 80           # dynamic 时忽略
auto_start = false
```

### Rust 类型定义

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ForwardRule {
    pub id: String,
    pub name: String,
    pub kind: ForwardKind,
    pub local_port: u16,
    pub remote_host: Option<String>,  // dynamic 时为 None
    pub remote_port: Option<u16>,     // dynamic 时为 None
    pub auto_start: bool,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum ForwardKind {
    Local,    // ssh -L
    Remote,   // ssh -R
    Dynamic,  // ssh -D，SOCKS5
}
```

```
┌─ sush ─ Port Forwards ──────────────────────────────┐
│ ┌──────────────┬───────────────────────────────────┐ │
│ │ prod-web-01● │  ● Web Admin  L 8080→localhost:80 │ │

│ │              │                                   │ │
```

- 左侧：有转发规则的主机列表；主机行聚合显示状态（● = 至少一条规则运行中）

- `Tab`：切换左右面板焦点
- `Enter`：在规则列表中切换启动/停止
- `q` 或 `Esc`：返回主界面

## ProxyJump 实现

单跳连接顺序（russh 层）：

```
1. russh 连接跳板机（使用跳板机的 Host 配置认证）
2. 在跳板机 session 上打开 direct-tcpip channel → 目标主机:22
3. 在该 channel 上再起一层 russh 客户端握手
4. 得到目标主机的 SSH handle，在其上建转发 channel
```

跳板机引用：`proxy_jump` 字段存储另一台主机的 `alias`，daemon 启动时解析为对应 `Host` 配置。若引用的主机不存在，规则状态设为 `Error("proxy jump host not found")`。

## Daemon 状态机

每条转发规则独立维护：

```
Stopped → Connecting → Running
                          │
                     SSH 断开 → Reconnecting（指数退避 1s→2s→4s→...→30s）
                                     │
                              重试超过 5 次 → Error(msg)
                                     │
                              用户手动 start → 重置重试计数，回到 Connecting
```

## 错误处理

- 端口已被占用：规则状态设为 `Error("port 8080 already in use")`，不影响其他规则
- SSH 认证失败：状态设为 `Error("authentication failed")`，不自动重试（需用户检查配置）
- 跳板机不可达：状态设为 `Error("proxy jump unreachable: ...")`
- Daemon 崩溃：TUI 下次发命令时 socket 连接失败，自动重新 spawn daemon

## 平台支持

| 平台 | IPC 机制 | 状态 |
|------|----------|------|
| macOS | Unix socket | v0.8 支持 |
| Linux | Unix socket | v0.8 支持 |
| Windows | 推迟（TCP localhost fallback） | v1.0 支持 |

## 不在 v0.8 范围内

- 多跳 ProxyJump（A → B → C → 目标）
- 转发流量统计（字节数、连接数）
- 转发规则导入/导出
- Windows daemon 支持
- 跨重启持久化重试状态
