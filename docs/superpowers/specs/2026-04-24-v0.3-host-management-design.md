# 设计文档：v0.3 主机管理 CRUD

## 背景

v0.1/v0.2 仅支持从 `~/.ssh/config` 自动导入主机，用户无法在 TUI 内新建、编辑或删除主机。v0.3 补全主机管理的完整闭环，同时将 SSH config 导入改为用户主动触发的操作。

## 功能范围

- TUI 内新建/编辑主机（全屏编辑视图）
- 主机删除（确认弹窗）
- 标签内联 chip 编辑器（带跨主机补全）
- 描述字段展示与编辑
- SSH config 导入改为手动触发 + 首次启动询问

## 状态机扩展

### 新增 AppMode 变体

```rust
pub enum AppMode {
    Main,
    Ssh,
    Sftp,
    Edit(EditTarget),       // 主机编辑全屏视图
    ImportSshConfig,        // SSH config 导入选择视图
}

pub enum EditTarget {
    New,
    Existing(String),       // host.id
}
```

### 模式转换规则

```
Main ──n──────────→ Edit(New)
Main ──e──────────→ Edit(Existing(id))
Main ──d──────────→ [ConfirmDialog overlay]（不切换 AppMode）
Main ──i──────────→ ImportSshConfig
Main 首次启动弹窗 → ImportSshConfig（用户确认后）

Edit ──ESC────────→ Main（丢弃草稿）
Edit ──Ctrl-S─────→ Main（保存）

ImportSshConfig ──Enter──→ Main（导入选中项）
ImportSshConfig ──ESC────→ Main（不导入）
```

### 主界面快捷键新增

```
n:New  e:Edit  d:Delete  i:Import SSH config
```

## 编辑视图（AppMode::Edit）

### 布局

```
┌─ New Host / Edit Host ──────────────────────────┐
│                                                  │
│  Alias        │ prod-web-01█                     │
│  Hostname     │ 192.168.1.10                     │
│  Port         │ 22                               │
│  User         │ deploy                           │
│  Identity     │ ~/.ssh/id_ed25519                │
│  Proxy Jump   │                                  │
│  Tags         │ [web] [prod] █                   │
│  Description  │ Main production web server       │
│                                                  │
│  Ctrl-S:Save  ESC:Cancel                        │
└──────────────────────────────────────────────────┘
```

### 字段交互

- `↑`/`↓` 或 `Tab`/`Shift-Tab` 在字段间切换，当前行高亮
- 文本字段：直接编辑，`←`/`→` 移动光标，`Backspace` 删字符，`Home`/`End` 跳首尾
- Port 字段：只允许数字字符输入
- Tags 字段：独立 chip 编辑器（见下节）

### 保存验证

| 规则 | 错误提示 |
|------|----------|
| Alias 不能为空 | "Alias is required" |
| Hostname 不能为空 | "Hostname is required" |
| Alias 不能与其他主机重复（新建时） | "Alias already exists" |
| Port 范围 1–65535 | "Port must be between 1 and 65535" |

验证失败时在视图顶部显示错误提示，不退出编辑视图。

### 新建主机的 ID 生成

`id = alias`（与 SSH config 导入保持一致）。若用户修改 Alias，`id` 随之更新（仅限新建时）；编辑已有主机时 `id` 不变。

## 标签 Chip 编辑器

### 光标模型

光标位于 tag 之间，tag 是不可进入的原子单元：

```
|[web] [prod] [nginx]       ← 光标在最左侧
[web]| [prod] [nginx]       ← 光标在 web 和 prod 之间
[web] [prod] [nginx]|       ← 光标在最右侧（输入区）
[web] [prod]| ngi█          ← 正在输入新 tag，光标在 prod 后
```

### 键盘行为

| 场景 | 按键 | 动作 |
|------|------|------|
| 无输入时 | `←` | 光标向左跳过一个 tag |
| 无输入时 | `→` | 光标向右跳过一个 tag |
| 无输入时 | `Backspace` | 删除光标左侧的 tag |
| 无输入时 | 任意可打印字符 | 在当前光标位置开始输入新 tag |
| 输入中 | 字符输入 | 过滤补全候选列表 |
| 输入中 | `↑`/`↓` | 在补全候选列表中移动 |
| 输入中 | `Enter`/`Tab` | 选中高亮候选或创建新 tag，光标移到新 tag 右侧 |
| 输入中 | `Backspace` | 删除输入最后一个字符；输入为空时删除光标左侧 tag |
| 输入中 | `ESC` | 取消当前输入，恢复光标位置 |

tag 无选中状态，只能整体删除，不能进入内部编辑。

### 补全候选

- 来源：所有主机的 `tags` 字段去重合并，按字母排序
- 过滤：排除当前编辑列表中已有的 tag
- 候选列表显示在 Tags 行上方，最多显示 5 条

### 失焦行为

Tags 字段失焦时，若有未提交的输入文本，自动创建为新 tag。

## 删除确认弹窗

主界面按 `d` 时，在当前视图上叠加 `ConfirmDialog`（不切换 AppMode）：

```
┌─────────────────────────────────────┐
│  Delete "prod-web-01"?              │
│  This only removes it from sush.    │
│                                     │
│  y:Confirm  n/ESC:Cancel            │
└─────────────────────────────────────┘
```

- `SshConfig` 来源的主机也可删除，副标题提示"only removes it from sush"
- `y` → 从 `hosts.toml` 删除，保存，关闭弹窗
- `n`/`ESC` → 关闭弹窗，不做任何修改

## SSH Config 导入

### 自动导入逻辑移除

删除 `App::new()` 中的 hash 对比与 `merge_ssh_config_hosts` 自动调用。`Metadata.ssh_config_hash` 字段保留但不再写入新值（兼容旧配置文件反序列化）。

### Metadata 结构更新

```rust
pub struct Metadata {
    pub ssh_config_hash: String,   // 保留，不再更新
    pub import_prompted: bool,     // 新增：是否已完成首次导入询问
}
```

### 首次启动询问流程

触发条件：`hosts` 为空 **且** `metadata.import_prompted == false` **且** `~/.ssh/config` 存在并有可解析主机。

主界面渲染完成后立即弹出确认弹窗：

```
┌──────────────────────────────────────┐
│  Found N hosts in ~/.ssh/config.     │
│  Import them into sush?              │
│                                      │
│  y:Yes   n/ESC:No                    │
└──────────────────────────────────────┘
```

- `y` → 关闭弹窗，进入 `ImportSshConfig` 视图
- `n`/`ESC` → 关闭弹窗，保存 `import_prompted = true`，留在主界面

从 `ImportSshConfig` 返回主界面时（无论导入还是 ESC），保存 `import_prompted = true`。

`~/.ssh/config` 不存在或无可解析主机时：跳过自动触发，`import_prompted` 不置位（下次启动若用户新增了 ssh config 仍会触发）。

### ImportSshConfig 视图布局

```
┌─ Import from ~/.ssh/config ─────────────────────┐
│                                                  │
│  [x] prod-web-01    192.168.1.10    deploy       │
│  [x] prod-db-01     192.168.1.20    deploy       │
│  [~] dev-server     10.0.0.5        lichen        │  ← 已存在，灰色不可选
│  [x] staging-web    10.0.1.10       deploy       │
│                                                  │
│  Space:Toggle  a:All  Enter:Import  ESC:Cancel   │
└──────────────────────────────────────────────────┘
```

- `↑`/`↓` 导航，`Space` 切换选中，`a` 全选/全取消可选项
- 已存在 sush 中的主机（按 `id` 匹配）显示为灰色 `[~]`，跳过键盘导航
- `Enter` 导入所有 `[x]` 项，追加到 `hosts.toml`，返回主界面
- `ESC` 不修改，返回主界面
