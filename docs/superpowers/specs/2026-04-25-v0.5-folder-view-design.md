# 设计文档：v0.5 虚拟目录导航

## 功能概述

用主机的 tag 字段实现虚拟文件夹结构。以 `/` 开头的 tag 被识别为路径 tag（如 `/prod`、`/prod/web`），sush 据此构建可导航的目录树。无路径 tag 的主机归入根目录 `/`。

主要功能：
- `f` 键进入全屏文件夹视图（`AppMode::FolderView`）
- 两列目录树 + 主机列表的三栏布局
- `j` 键快速跳转目录（模糊搜索 + Tab 补全）
- `/` 键在当前目录范围内模糊搜索主机

---

## 路径 Tag 规则

- 以 `/` 开头的 tag 为路径 tag：`/prod`、`/prod/web`、`/`（根）
- 普通 tag（如 `nginx`、`web`）不参与目录导航，仍用于模糊搜索
- 一台主机可以有多个路径 tag，同时出现在多个目录下
- 无路径 tag 的主机自动归入根目录 `/`
- `/` 与 `/xxx` 并列为一级目录，出现在左列

## 状态机

### 新增 AppMode 变体

```rust
pub enum AppMode {
    Main,
    Ssh,
    Sftp,
    Edit,
    ImportSshConfig,
    FolderView,
}
```

`f` 进入，`ESC` 退出回主界面。

### 数据结构

```rust
pub struct FolderNode {
    pub path: String,           // 完整路径，如 "/prod/web"
    pub name: String,           // 显示名，如 "web"
    pub children: Vec<String>,  // 子路径完整路径列表
}

pub struct FolderViewState {
    pub tree: HashMap<String, FolderNode>,
    pub col_a: Vec<String>,     // 左列目录路径列表
    pub col_b: Vec<String>,     // 右列目录路径列表（左列选中项的子目录）
    pub sel_a: usize,
    pub sel_b: usize,
    pub depth: usize,           // 当前视图层级（0 = 根级）
    pub focus: FolderFocus,
    pub jump: Option<JumpState>,
    pub search: Option<SearchState>, // / 键搜索状态
}

pub enum FolderFocus { DirA, DirB, Hosts }

pub struct JumpState {
    pub input: String,
    pub candidates: Vec<String>, // 匹配的目录路径，按模糊分数排序
    pub sel: usize,
}

pub struct SearchState {
    pub scope_path: String,  // 当前限定目录，如 "/prod"
    pub query: String,
}
```

### 目录树构建

进入 FolderView 时实时构建，不持久化：

```
1. 收集所有主机的路径 tag（以 / 开头的 tag）
2. 无路径 tag 的主机加入 "/" 节点
3. 对每个路径 tag，确保父路径节点存在（递归创建中间节点）
4. 构建 parent → children 映射
5. col_a 初始化为根级目录（深度 0 的所有节点）
6. col_b 初始化为 col_a[0] 的子目录
```

## 界面布局

三列比例：`[20%] [20%] [60%]`

```
┌─ Folders ──────────────────────────────────────────────┐
│                                                         │
│  /         ▶ web         prod-web-01  192.168.1.10     │
│  /prod     ▶ db          prod-web-02  192.168.1.11     │
│  /dev        cache       prod-db-01   192.168.1.20     │
│  /staging                                              │
│                                                         │
│  j:Jump  /:Search  Tab:Switch  Enter:Enter  ESC:Exit   │
└─────────────────────────────────────────────────────────┘
```

- **左列（DirA）**：当前层级目录，`▶` 表示有子目录
- **中列（DirB）**：左列选中项的子目录（无子目录时为空）
- **右列（Hosts）**：当前焦点目录下所有主机（递归），复用 `HostList` widget

### j 跳转浮层

```
┌─ Jump to ─────────────────┐
│  path:/prod/w█             │
├───────────────────────────┤
│ > /prod/web               │
│   /prod/web/frontend      │
└───────────────────────────┘
```

- 居中浮层，宽 50%，最多显示 8 条候选
- 候选列表对所有路径做模糊搜索（不限当前目录）

### / 搜索输入框

在左列或标题栏显示：

```
path:/prod > web█
```

- `path:/xxx` 前缀为只读，显示当前 scope
- `>` 后面是用户输入

## 键盘导航

### 焦点在左列 DirA

| 按键 | 动作 |
|------|------|
| `↑` / `↓` | 在左列目录间移动；右列自动更新为选中项的子目录 |
| `Enter` / `→` | 进入：左列变为右列内容，右列变为新选中项的子目录，深度 +1 |
| `Backspace` / `←` | 返回上一级，深度 -1；根级无效 |
| `Tab` | 焦点切换到 DirB（若有内容）或 Hosts |
| `j` | 打开跳转浮层 |
| `/` | 打开路径限定搜索 |
| `ESC` | 退出 FolderView |

### 焦点在右列 DirB

| 按键 | 动作 |
|------|------|
| `↑` / `↓` | 在右列目录间移动；Hosts 列实时更新 |
| `Enter` / `→` | 进入下一级 |
| `←` / `Shift-Tab` | 焦点返回 DirA |
| `Tab` | 焦点切换到 Hosts |

### 焦点在主机列表 Hosts

| 按键 | 动作 |
|------|------|
| `↑` / `↓` | 在主机间移动 |
| `Enter` | SSH 连接 |
| `s` | SFTP 连接 |
| `Tab` / `Shift-Tab` | 焦点返回 DirB（若有）或 DirA |

### 跳转浮层（JumpState 激活）

| 按键 | 动作 |
|------|------|
| 字符 | 追加到输入，模糊过滤候选 |
| `Backspace` | 删除最后一个字符 |
| `↑` / `↓` | 移动候选高亮 |
| `Tab` | 将高亮候选路径填入输入框（路径补全） |
| `Enter` | 跳转到高亮候选目录，更新两列显示，关闭浮层 |
| `ESC` | 取消，关闭浮层 |

### 路径限定搜索（SearchState 激活）

| 按键 | 动作 |
|------|------|
| 字符 | 追加查询，过滤当前 scope 下的主机 |
| `Backspace` | 删除最后一个字符 |
| `ESC` | 清空查询，恢复完整主机列表，不退出 FolderView |
| `Enter` | 保持当前过滤结果，关闭搜索输入 |

## 主机过滤逻辑

**当前焦点目录**：焦点在 DirA 时取 `col_a[sel_a]`，焦点在 DirB 时取 `col_b[sel_b]`，焦点在 Hosts 时取上一次的目录焦点。

**递归过滤**：

```
selected_hosts(path, hosts) =
  hosts where host has any path_tag t such that t == path || t.starts_with(path + "/")
  加上：若 path == "/" 则包含无路径 tag 的主机
```

**路径限定搜索叠加**：在递归过滤结果上再做模糊搜索（使用现有 `fuzzy::search`，history 参数传当前 ConnectionHistory）。

## 与现有功能的关系

- 连接历史与排序：FolderView 的主机列表也按 history 排序（复用 `fuzzy::search`）
- 连通性 probe：进入 SSH/SFTP 后 probe 状态照常工作
- 路径搜索（`/`）：仅在 FolderView 内有效；普通模糊搜索（主界面的搜索框）不受影响
- `j` 跳转的候选来源：所有路径节点，不限当前目录
