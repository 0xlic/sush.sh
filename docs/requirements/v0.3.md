# sush.sh (sush) v0.3 需求文档

## 版本目标

v0.3 的目标是补齐主机管理能力，使用户可以在 TUI 内完成主机的新建、编辑、删除和手动导入，不再依赖启动时自动合并作为主要入口。

## 变更范围

本版本在 `v0.2` 基础上新增主机管理和手动导入流程，并将 SSH Config 的使用方式从“自动合并为主”调整为“手动导入 + 首次启动引导”。

## 功能范围

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | 主机 CRUD | 在 TUI 内新建、编辑、删除主机 |
| F2 | 标签管理 | 编辑标签、按标签参与搜索 |
| F3 | 手动 SSH Config 导入 | 由用户显式触发导入 |
| F4 | 首次启动导入引导 | 首次运行时提示是否从 `~/.ssh/config` 导入 |

## 需求细则

### F1: 主机 CRUD

**规则**：
- 主界面提供新建、编辑、删除主机入口
- 主机表单至少包含：别名、地址、端口、用户、IdentityFile、ProxyJump、标签、描述
- 别名和地址为必填项，端口默认 `22`
- 保存时校验端口范围与基本字段完整性
- 删除主机前需要确认
- 删除只影响 sush 本地配置，不修改用户原始 `~/.ssh/config`

### F2: 标签管理

**规则**：
- 主机支持多个扁平标签
- 标签可在主机编辑表单中增删改
- 标签参与主界面模糊搜索
- 标签不是目录树，不做层级化含义

### F3: 手动 SSH Config 导入

**规则**：
- 主界面提供显式导入入口
- 导入列表展示来自 `~/.ssh/config` 的候选主机
- 已存在于 sush 配置中的主机应标记为已存在，避免重复导入
- 用户可逐项勾选后确认导入
- sush 已存在主机优先，不被 ssh config 中同名条目覆盖

### F4: 首次启动导入引导

**规则**：
- 首次启动且本地主机列表为空时，若 `~/.ssh/config` 中存在候选主机，则提示是否导入
- 用户拒绝后不应在每次启动时重复打扰
- 导入提示状态保存在本地 metadata 中

## 验收重点

- 用户无需改配置文件即可在 TUI 中维护主机
- 手动导入流程可区分“新增可导入”和“已存在”主机
- 首次启动引导只在合适时机出现一次
- 保存和删除后主机列表立即刷新并持久化
