# sush.sh (sush) v0.1 需求文档

## 版本目标

v0.1 是 MVP 版本，目标是在同一个终端工具里打通主机搜索、SSH 连接、SFTP 浏览和基础文件传输，建立 `SSH ↔ SFTP` 的统一使用路径。

## 功能范围

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | SSH Config 自动导入 | 首次启动时自动读取 `~/.ssh/config` 并生成主机列表 |
| F2 | 主界面搜索与主机列表 | 支持搜索、导航、进入 SSH / SFTP |
| F3 | SSH 接管模式 | 连接后进入接管模式，支持前缀键切换 |
| F4 | 单面板 SFTP 浏览器 | 进入目录、返回上级、切换本地 / 远程视图 |
| F5 | 文件上传 / 下载 | 基础单文件传输与进度展示 |
| F6 | 双 `Ctrl-C` 返回主界面 | 在 SFTP 中快速返回主界面 |

## 需求细则

### F1: SSH Config 自动导入

**目标**：首次启动后直接看到可连接主机，减少手工录入。

**规则**：
- 自动读取 `~/.ssh/config`
- 解析字段：`Host`、`HostName`、`User`、`Port`、`IdentityFile`、`ProxyJump`
- 从每个 `Host` 条目中选择第一个非通配符、非反选的 pattern 作为主机别名；若不存在则跳过该条目
- `HostName` 缺失时回退为别名，`Port` 缺失时默认 `22`
- 支持多个 `IdentityFile` 声明
- `ProxyJump` 仅使用第一个值
- 启动时检测 `~/.ssh/config` 变化；只新增 sush 中不存在的主机，不覆盖已有主机
- `~/.ssh/config` 中已删除的主机不会自动从 sush 中移除
- 导入的主机标记来源为 `ssh_config`，与用户手动添加的主机区分

**验收标准**：
- 首次启动后可看到可导入的主机条目
- 修改 ssh config 后重启，新增主机会被补充导入
- 已存在主机不会被 ssh config 中的变更自动覆盖

### F2: 主界面搜索与主机列表

**目标**：用最小界面快速定位并连接主机。

**规则**：
- 顶部提供搜索输入框，底部提供快捷键提示栏
- 搜索支持匹配主机别名、HostName / IP、User、标签、描述
- 输入即搜，空搜索显示全部主机
- 主机列表显示别名、地址、标签
- 支持上下键导航，选中行高亮
- 主界面默认快捷键包含 `/:搜索  Enter:SSH  s:SFTP  q:退出`

### F3: SSH 接管模式

**目标**：在 MVP 阶段先保证 SSH 会话能稳定工作。

**规则**：
- 选中主机后按 `Enter` 建立 SSH 连接
- 认证顺序为 `ssh-agent -> IdentityFile -> 密码认证`
- 密码认证通过 TUI 弹窗输入
- 连接成功后进入接管模式，将本地输入转发到远程 PTY
- 远程标准输出 / 标准错误输出直接显示到终端
- 兼容常见交互式程序，如 `vim`、`tmux`、`htop`
- `exit` / `Ctrl-D` 退出远程 shell 后自动返回主界面
- 前缀键 `Ctrl-\` 用于在 SSH 与 SFTP 间切换

### F4: 单面板 SFTP 浏览器

**目标**：在同一条 SSH 连接中浏览文件并进行基础传输。

**规则**：
- 按 `s` 或在 SSH 中按 `Ctrl-\` 进入 SFTP
- v0.1 采用单面板浏览器，同一时刻只显示一个面板
- 支持本地 / 远程两种视图，并可切换当前视图
- 默认打开远程用户 `$HOME`，失败时回退到 `/`
- 本地视图默认打开当前工作目录 `PWD`
- 目录排前，文件排后，各自按名称排序
- 显示目录标记和人类可读文件大小
- `Enter` 进入目录，`..` 返回上级目录

### F5: 文件上传 / 下载

**目标**：支持最基本的单文件传输。

**规则**：
- 远程视图中选中文件后按 `d` 下载到本地当前目录
- 本地视图中选中文件后按 `u` 上传到远程当前目录
- 仅要求支持普通文件的单次上传 / 下载
- 传输过程中展示当前文件进度
- 传输完成后刷新当前视图

### F6: 双 `Ctrl-C` 返回主界面

**规则**：
- 在 SFTP 视图下，500ms 内双击 `Ctrl-C` 返回主界面
- 在 SSH 接管模式下，`Ctrl-C` 透传给远程进程，不被拦截
- 在主界面中，搜索框焦点下 `Ctrl-C` 清空搜索；主机列表焦点下 `Ctrl-C` 退出程序

## 非功能要求

- 启动时间目标 `< 200ms`
- 搜索响应目标 `< 50ms`
- 空闲内存目标 `< 30MB`
- 分发形式为单文件二进制，无额外运行时依赖
- 最低终端要求为支持 256 色和 UTF-8
