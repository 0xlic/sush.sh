# sush.sh (sush) v0.7 需求文档

## 版本目标

v0.7 聚焦文件操作体验，补齐目录递归传输、远程文件编辑、双面板 SFTP、多选批量操作、后台传输队列和最小断点续传能力。

## 变更范围

本版本在 `v0.6` 基础上扩展 SFTP 视图与传输运行时，是一个以文件操作为核心的综合增强版本。

## 功能范围

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | 自适应双面板 SFTP | 宽终端同屏显示本地 / 远程，窄终端回退单面板 |
| F2 | SFTP 多选批量操作 | 支持单选、区间选择、批量传输与批量删除 |
| F3 | 递归目录传输 | 保留所选目录本身并显示聚合进度 |
| F4 | 远程文件编辑 | 默认应用打开，保存后自动回传 |
| F5 | 传输队列与后台传输 | 当前连接范围内单一 FIFO 队列 |
| F6 | 最小断点续传 | 普通文件按目标端已存在大小续传 |

## 需求细则

### F1: 自适应双面板 SFTP

**规则**：
- 宽终端下固定 `50/50` 显示本地与远程两个面板
- 窄终端下回退到只显示当前激活面板
- `Tab` 只切换焦点，不重置两侧各自的选中项
- 本地与远程面板各自维护独立状态
- 目录排前、文件排后，各自按名称排序

### F2: SFTP 多选批量操作

**规则**：
- 本地 / 远程面板分别维护独立多选集合和锚点
- 单击 `Space` 切换当前项选中状态并更新锚点
- 在另一行上快速双击 `Space`，选中锚点到当前项之间的闭区间
- `Esc` 只清空当前激活面板的多选状态
- 本地多选时按 `u` 批量上传；远程多选时按 `d` 批量下载；`D` 用于批量删除
- 操作成功后刷新对应面板并清空当前激活面板的多选状态

### F3: 递归目录传输

**规则**：
- 本地或远程选中目录时，支持递归上传 / 下载
- 递归传输保留所选目录本身，而不是只展开其子内容
- 先准备目录结构，再逐文件顺序传输
- 进度展示为聚合语义，至少包含 `当前文件序号 / 总文件数`
- 当前实现为单连接范围内的单任务串行执行，不做并行传输

### F4: 远程文件编辑

**规则**：
- 仅在远程视图下对普通文件生效，触发键为 `e`
- 先把远端文件下载到本地临时工作区，再用系统默认图形化应用打开
- 平台打开方式：
  - macOS：`open`
  - Linux：`xdg-open`
  - Windows：系统默认文件关联
- 本地文件保存后自动检测内容变化并回传
- 若内容与上次成功上传版本相同，不重复上传
- 上传失败时保留会话，后续内容变化后继续重试
- 自动上传时先写远端临时文件；必要时先备份旧文件，再切换到目标路径
- 存在远程编辑会话时，退出 SFTP 前需先处理当前编辑会话

### F5: 传输队列与后台传输

**规则**：
- 上传、下载、递归上传、递归下载都进入当前连接范围内的同一 FIFO 队列
- 同一时间只运行一个活动任务，当前任务结束后自动启动下一个
- 离开 SFTP 返回主页或 SSH 时，后台队列继续执行
- 真正断开当前 SSH 连接时，清空该连接的传输队列与运行态
- 主页面、SSH、SFTP 底部右侧统一显示紧凑传输 badge，例如 `↑ 2/10 37%` 或 `↓ 2/10 37%`
- badge 只显示方向、队列位置与当前文件百分比，不显示 ETA、速率或文件名

### F6: 最小断点续传

**规则**：
- 仅普通文件支持断点续传
- 下载时以本地目标文件已有大小作为续传偏移；上传时以远端目标文件已有大小作为续传偏移
- 只有当目标侧大小 `<=` 源文件大小时才允许续传
- 目标侧大小大于源文件大小时，回退为从 `0` 全量重传
- 递归传输和后台队列逐文件复用同一套单文件 helper
- 不做内容哈希校验，也不保存跨重启续传记录

## 验收重点

- SFTP 在宽终端和窄终端下都能保持明确、稳定的交互
- 批量传输、递归传输和后台队列可以协同工作，不互相打断
- 远程文件编辑能做到“打开即改，保存即回传”
- 普通文件重复传输时可按大小进行最小续传
