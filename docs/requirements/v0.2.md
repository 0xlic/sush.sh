# sush.sh (sush) v0.2 需求文档

## 版本目标

v0.2 的目标是把 SSH 模式从“接管整个终端”升级为“嵌入式终端模拟器”，使 sush 的 TUI 在 SSH 会话期间持续可见。

## 变更范围

本版本在 `v0.1` 基础上，只调整 SSH 会话的呈现方式，不改变主机管理、SFTP 和基础传输的核心链路。

## 需求细则

### F1: 嵌入式终端模拟器

**目标**：让 SSH 会话显示在 TUI 内部，而不是切走整个界面。

**规则**：
- 集成终端模拟器状态机，正确处理常见 VT100 / xterm 输出
- 远程 PTY 输出字节流写入终端模拟器，再由 ratatui 渲染
- 连接期间 sush 界面持续存在，不再依赖切换 alternate screen
- 支持常见全屏交互程序，如 `vim`、`tmux`、`htop`
- 终端 resize 事件继续同步到远程 PTY

### F2: SSH 状态栏常驻

**规则**：
- SSH 模式下底部始终显示主机名和快捷键提示
- 终端内容显示在主区域，状态栏固定在底部一行
- 状态栏至少包含 `Ctrl-\:SFTP` 和 `Ctrl-D:断开`

### F3: 前缀键仍在 TUI 层处理

**规则**：
- `Ctrl-\` 仍作为 sush 的模式切换前缀键
- 除前缀键外，其他按键保持直通远程会话
- 从 SSH 切到 SFTP 后，再次按 `Ctrl-\` 能回到原 SSH 会话
- 切回 SSH 时应保留已有终端渲染内容

## 验收重点

- SSH 模式下界面不闪屏、不离开 TUI
- 连接主机后可正常执行命令并看到渲染结果
- `Ctrl-\` 在 SSH / SFTP 间来回切换时，会话不中断
- 终端窗口缩放后，远程 shell 的列宽和行高同步更新
