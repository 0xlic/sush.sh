# sush.sh (sush) v0.5 需求文档

## 版本目标

v0.5 的目标是在主界面引入“虚拟目录导航”，用 path 类型标签把主机组织成目录结构，降低大规模主机列表的浏览成本。

## 变更范围

本版本在 `v0.4` 基础上新增目录栏、目录跳转和 `path:` 范围搜索，不改变 SSH / SFTP 的核心行为。

## 需求细则

### F1: path 类型标签

**规则**：
- 以 `/` 开头的标签视为路径标签，例如 `/prod`、`/prod/web`
- 普通标签仍参与搜索，但不参与目录树构建
- 一台主机可以拥有多个路径标签，因此可同时出现在多个目录下
- 没有路径标签的主机归入根目录 `/`

### F2: 主界面目录栏

**规则**：
- 主界面可显示 / 隐藏左侧目录栏
- 目录栏展示由 path 标签构建的目录树
- 目录栏显示时，主机列表只展示当前目录范围内的主机
- 切换目录不会修改主机数据，只改变可见范围
- 目录栏与主机列表之间支持焦点切换

### F3: 目录导航

**规则**：
- 支持在目录树中上下移动
- `Enter` / `→` 进入子目录
- `Backspace` / `←` 返回上一级目录
- `j` 打开目录跳转浮层，并在确认后跳到目标目录
- 目录栏聚焦时，右侧主机列表只作预览，不承载主机操作焦点

### F4: `path:` 范围搜索

**规则**：
- 当目录栏显示时，搜索框应体现当前目录范围
- 搜索输入可带 `path:` 前缀，用于表达目录范围内搜索
- 搜索结果必须限制在当前目录对应的主机集合中
- 从目录栏切回主机列表后，搜索状态保持一致

## 验收重点

- 使用 path 标签后，主机可按目录方式浏览和定位
- 目录栏显示与隐藏不会破坏现有主机搜索体验
- `j` 目录跳转和 `path:` 搜索可在大列表场景中明显减少操作步数
