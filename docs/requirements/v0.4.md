# sush.sh (sush) v0.4 需求文档

## 版本目标

v0.4 聚焦主界面使用体验，补齐“最近连接历史”和“主机连通性探测”，让用户更快定位常用主机并提前感知网络状态。

## 变更范围

本版本在 `v0.3` 基础上新增历史记录、搜索排序加权和选中主机的 TCP 连通性探测。

## 需求细则

### F1: 连接历史记录

**规则**：
- 每次成功进入 SSH 或 SFTP 时记录该主机最近连接时间
- 历史记录独立存储，不混入主机配置
- 当搜索词为空时，主机列表优先按最近连接时间倒序展示
- 未连接过的主机在无搜索词时按主机地址或名称稳定排序

### F2: 搜索结果近期加权

**规则**：
- 搜索结果仍以模糊匹配为主
- 当多个候选主机匹配分值相近时，近期连接过的主机应排名更靠前
- 加权只影响排序，不改变匹配集合

### F3: 选中主机 TCP 连通性探测

**规则**：
- 在主界面主机列表中，选中某台主机并停留约 1 秒后，自动触发 TCP probe
- 探测目标为主机配置中的 `hostname:port`
- 探测结果使用描述行中的圆点状态显示：
  - 灰色：探测中
  - 绿色：可连接
  - 红色：不可连接
- 光标切走后，上一台主机的探测状态应立即失效
- 快速切换选中项时不应频繁触发无意义探测

### F4: 连接错误短时反馈

**规则**：
- SSH 或 SFTP 连接失败时，错误信息在主界面描述行显示
- 错误信息使用明显的失败样式
- 若后续无新错误，显示会在短时间后自动恢复为正常描述信息

## 验收重点

- 常用主机在无搜索词和模糊搜索场景下更容易排到前面
- 停留在主机列表项上可以看到明确的连通性状态
- 错误反馈不打断主流程，但用户可直接感知失败原因
