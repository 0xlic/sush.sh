# sush.sh (sush) v0.8 需求文档

## 版本目标

v0.8 的目标是增加高级网络能力，包括 SSH 端口转发管理、单跳 ProxyJump、动态转发（SOCKS5）以及基于 daemon 的转发运行时管理。

## 变更范围

本版本在 `v0.7` 基础上新增 `tunnel/` 模块、守护进程、转发规则配置和转发管理视图，不改变普通 SSH / SFTP 的核心链路。

## 功能范围

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | 转发规则配置 | 主机可持久化本地 / 远程 / 动态转发规则 |
| F2 | 转发管理器 | 主界面按 `p` 进入 host / rule 双面板管理视图 |
| F3 | daemon 运行时 | 独立守护进程持有转发连接与状态 |
| F4 | 单跳 ProxyJump | 转发支持经跳板机建立 SSH |
| F5 | 转发状态模型 | 支持运行、重连、错误等状态展示 |

## 需求细则

### F1: 转发规则配置

**规则**：
- 转发规则挂在具体主机之下持久化保存
- 规则类型包括：
  - `Local`：本地端口转发
  - `Remote`：远程端口转发
  - `Dynamic`：动态转发（SOCKS5）
- 每条规则至少包含名称、类型、本地端口、自动启动开关
- 对于 `Local` 和 `Remote` 规则，还需包含远端主机和远端端口
- `Dynamic` 规则不需要远端主机和远端端口字段

### F2: 转发管理器

**规则**：
- 主界面按 `p` 进入转发管理视图
- 视图采用双面板：左侧展示主机，右侧展示当前主机的转发规则
- 支持新建、编辑、删除规则
- 规则列表中展示规则类型、路由信息和当前状态
- 在规则列表中按 `Enter` 可启动或停止对应规则

### F3: daemon 运行时

**规则**：
- 转发运行时由独立 daemon 进程持有，而不是绑定在当前 TUI 生命周期内
- daemon 在 Unix 平台监听 `~/.config/sush/daemon.sock`
- TUI 通过 IPC 与 daemon 通信，至少支持 `status`、`start`、`stop`
- daemon 需要能在 TUI 启动时被自动拉起或复用现有实例
- daemon 在退出时清理运行时文件，如 socket 和 pid 文件

### F4: 单跳 ProxyJump

**规则**：
- v0.8 只支持单跳 ProxyJump，不支持多跳链式转发
- 当主机配置了 `proxy_jump` 时，daemon 按 alias 解析跳板机主机
- 建链流程为：先连接跳板机，再通过 `direct-tcpip` 建立到目标主机的第二段 SSH
- 跳板机缺失、认证失败、关键参数缺失等情况视为致命错误，不进入重试退避

### F5: 转发状态模型

**规则**：
- 每条规则在 daemon 内维护独立状态：
  - `Stopped`
  - `Connecting`
  - `Running`
  - `Reconnecting`
  - `Error`
- `auto_start` 规则在 daemon 启动后自动进入连接流程
- 可重试错误按退避策略自动重连
- 端口占用、跳板机不存在、认证失败等致命错误直接进入 `Error`
- TUI 必须周期性刷新状态并显示最新结果

## 验收重点

- 用户可以在 TUI 内集中管理转发规则，而不必手工记忆命令
- TUI 退出后，daemon 持有的转发仍可继续运行
- 单跳 ProxyJump 和动态转发可作为规则统一管理
- 错误、重连和运行状态在视图中清晰可见
