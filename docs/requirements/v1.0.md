# sush.sh (sush) v1.0 需求文档

## 版本目标

v1.0 的目标不是继续铺新功能面，而是把已有能力整理为可稳定发布、可分发、可验证的正式版本。

## 范围说明

v1.0 在 `v0.8` 已有能力基础上，以发布验收为主，重点覆盖测试和分发。

## 功能范围

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | macOS 平台测试 | 核心能力在 macOS 手动 smoke test 验证通过；Linux / Windows 留待后续版本 |
| F2 | 二进制发布 | GitHub Actions + cargo-dist 自动构建六平台二进制包并发布到 GitHub Releases |
| F3 | 文档完整性 | README、版本需求文档、架构文档、路线图之间口径一致 |

## 需求细则

### F1: macOS 平台测试

**规则**：
- 在 macOS 上手动验证以下核心能力：
  - 主机管理（新建、编辑、删除、导入）
  - SSH 连接与 SFTP 切换
  - 文件传输（上传、下载、文件夹传输、断点续传）
  - 远程文件编辑与自动上传
  - 系统钥匙串凭证存储与读取
  - 端口转发管理（新建规则、启动、停止、状态刷新）
- Linux 和 Windows 平台测试不在 v1.0 范围内，待后续版本补充

### F2: 二进制发布

**规则**：
- 通过 GitHub Actions + cargo-dist 在 tag 推送后自动触发构建
- 产出六个平台的二进制包：
  - macOS arm64 / x86_64
  - Linux arm64 / x86_64
  - Windows x86 / x86_64
- `Cargo.toml` 的 `version` 字段必须与 tag 一致后再推送
- Release 产物自动上传到 GitHub Releases

### F3: 文档完整性

**规则**：
- README（中英文）、版本需求文档、架构文档、路线图之间口径一致
- README 中的功能描述与快捷键表覆盖 v0.1–v0.8 的所有已发布能力

## 发布验收清单

- macOS 核心功能 smoke test 通过
- 推 tag 后 GitHub Actions 成功构建并发布 Release
- 文档与行为一致，无遗漏的主要功能
