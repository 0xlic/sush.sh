# sush.sh (sush) v1.0 需求文档

## 版本目标

v1.0 的目标不是继续铺新功能面，而是把已有能力整理为可稳定发布、可分发、可验证的正式版本。

## 范围说明

v1.0 在 `v0.8` 已有能力基础上，以发布验收为主，重点覆盖测试、分发、安装和文档完整度。

## 功能范围

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | 全平台测试覆盖 | 核心能力在 macOS / Linux / Windows 验证通过 |
| F2 | 发布分发 | 提供 Homebrew / AUR / Scoop / cargo install 等入口 |
| F3 | 文档完善 | 用户文档、README、man page 与当前行为一致 |
| F4 | 错误与边界收敛 | 补齐主要失败路径和边界场景处理 |
| F5 | 性能与稳定性 | 对启动、搜索、内存和长时间使用稳定性做验收 |

## 需求细则

### F1: 全平台测试覆盖

**规则**：
- 至少覆盖 macOS、Linux、Windows 三个平台
- 需验证主机管理、SSH 连接、SFTP、文件传输、远程编辑、端口转发等核心能力
- 对平台特有能力需分别验证，如系统钥匙串和默认应用打开行为
- 测试结果应能支撑正式发布判断

### F2: 发布分发

**规则**：
- 提供面向用户的安装入口：
  - Homebrew
  - AUR
  - Scoop
  - `cargo install`
- 分发物应与 `Cargo.toml` 版本一致
- Release 产物命名和平台映射应保持稳定可识别

### F3: 文档完善

**规则**：
- README、版本需求文档、架构文档、路线图之间口径一致
- 提供可直接上手的用户文档
- 提供 `man page` 或等价命令行帮助入口
- 文档中不保留与当前行为不一致的历史描述

### F4: 错误与边界收敛

**规则**：
- 对主要失败路径提供清晰、可恢复或可理解的提示
- 覆盖典型边界场景，如连接失败、权限不足、端口冲突、文件冲突、系统能力缺失
- 避免因单点错误导致 TUI 状态错乱或数据损坏

### F5: 性能与稳定性

**规则**：
- 启动、搜索和空闲内存表现至少不劣于前序版本目标
- 长时间运行 SSH、SFTP、后台传输和转发时保持稳定
- 在窗口缩放、视图切换、网络抖动等场景下不出现明显回归

## 发布验收清单

- 核心功能在三平台 smoke test 通过
- 安装分发入口可用
- 文档与行为一致
- 无阻塞发布的高优先级已知问题
